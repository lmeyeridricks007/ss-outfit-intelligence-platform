# Feature Spec Audits

Audits verify that feature specs meet depth and implementation-readiness bar. Each audit checks:

- Depth is sufficient  
- Feature is not still too abstract  
- Interactions with other modules are clear  
- APIs / events / data are sufficiently specified  
- UI and backend implications are covered  
- Implementation teams could act on the document  

**Verdict:** Pass | Pass with minor improvements | Needs revision  

**Naming:** `{feature-slug}-audit.md` (e.g. `delivery-api-audit.md`).
