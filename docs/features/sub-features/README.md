# Sub-feature capability specifications

This directory contains the feature-stage capability breakdown generated from the feature deep-dives in `docs/features/`.

## Layout

- One folder per feature under `docs/features/sub-features/<feature-name>/`
- One markdown file per sub-feature capability inside each feature folder
- Portfolio audit artifacts under `docs/features/sub-features/audits/`

## Source of truth

- `docs/features/feature-spec-index.md`
- the parent feature deep-dives in `docs/features/*.md`
- `docs/project/` canonical standards and rubrics
- `docs/features/open-decisions.md` for unresolved cross-cutting items
