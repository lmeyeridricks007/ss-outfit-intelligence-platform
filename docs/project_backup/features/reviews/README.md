# Feature Spec Reviews

Reviews are stored here per the prompt requirements. Each review includes:

1. Overall Assessment  
2. Strengths  
3. Missing Business Detail  
4. Missing Workflow Detail  
5. Missing Data / API Detail  
6. Missing UI Detail  
7. Missing Integration Detail  
8. Missing Edge Cases  
9. Missing Implementation Detail  
10. Suggested Improvements  
11. Scorecard  
12. Confidence Rating  
13. Recommendation  

**Review categories:** clarity, completeness, functional depth, technical usefulness, cross-module consistency, implementation readiness.

**Threshold:** Every score ≥ 9/10; confidence ≥ 95% for promotion. Below threshold → apply suggested improvements and re-review.

**Naming:** `{feature-slug}-review.md` (e.g. `delivery-api-review.md`).
